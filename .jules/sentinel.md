## 2025-02-20 - Missing Path Validation on Local File Upload
**Vulnerability:** The `uploadZulipFile` function in `src/zulip/client.ts` was taking a file path (`filePath`) and uploading it without validating its location. This allowed arbitrary file exfiltration (Path Traversal) if an attacker could control the `filePath` parameter, potentially exposing sensitive files outside the intended directories (`os.tmpdir()` and `dataDir`).
**Learning:** Security hardening must be enforced systematically across utility functions, especially those interacting with the file system and uploading data to external destinations. Even if callers are expected to pass safe paths, defense-in-depth requires validation within the critical function itself.
**Prevention:** Always validate that local file paths provided as parameters fall strictly within authorized directories using `path.resolve()` and checking directory prefixes (`startsWith(authorizedDir + path.sep)`) before performing sensitive operations like file reads or uploads.
